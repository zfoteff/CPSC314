NoteStore

Zac Foteff

Notetaking applications are rigid in their design by limiting users to only creating text or basic lists in their documents. I have created a web app that allows users to not only create notes, but also create sections of notes that can act as a repository of encrypted documents. Users will be able to create separate accounts with their own segregated notes and documents. Users will be able to login to their account, create new notes, and upload/download files in their repositories. Functional Requirements:

  1. Users must be able to sign in / create new user

  2. User must be able signout and switch accounts from any screen

  3. Users must be able to view all documents and sections associated with their account

  4. Users must be able to toggle between read and write modes for notes page

  5. User must be able to share document's from the webpage using email

  6. User documents must be retrieved on sign in

  7. User must be able to retrieve documents linked to their account

  8. User's must be able to upload encrypted documents into the database

  9. User documents must be decrypted before being served to used

  10. User must be able to make changes to their account through the settings page
